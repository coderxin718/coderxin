# mctpweb —— 多模态作物表型数据采集与在线处理平台

## 项目背景

在数字农业与精准育种领域，作物表型（phenotype）的高通量获取是核心瓶颈之一。传统的人工测量方式效率低下、主观性强，难以满足大规模育种筛选的需求。**mctpweb** 是我开发的一套面向作物表型研究的 Web 平台，能够同时采集和处理 RGB 图像、高光谱成像（HSI）、激光雷达（LiDAR）点云和热红外成像四种模态的数据，在线完成从原始数据到生理生化参数的一站式分析。

## 系统架构

项目采用前后端分离架构，配合桌面端数据采集工具，形成完整的数据管道：

```
桌面采集端 (WPF)  ──SocketIO──▶  Flask 后端  ◀──HTTP/REST──▶  Vue3 前端
                                     │
                                     ├── MySQL (元数据 + 处理结果)
                                     ├── 文件系统 (原始数据存储)
                                     └── RabbitMQ + Celery (异步任务，已封存)
```

| 层 | 技术选型 |
|---|---|
| **Web 后端** | Python Flask + Flask-SocketIO + SQLAlchemy ORM |
| **异步任务** | Celery + RabbitMQ（大规模批处理，当前版本已封存） |
| **前端** | Vue 3 + TypeScript + Vite + Element Plus + Pinia 状态管理 |
| **可视化** | ECharts（图表） + Three.js（3D 点云） + Fabric.js（图像标注） |
| **桌面采集端** | C# WPF (.NET Framework 4.8) + SocketIO.Client |
| **数据库** | MySQL |

## 数据层级模型

系统采用层次化的数据组织方式，贴合农业田间试验的实际场景：

```
用户 (User)
 └── 项目 (Project)          —— 例如 "2024年冬小麦试验"
      └── 任务 (Task)         —— 例如 "拔节期-第一次测量"
           ├── 校正文件        —— 高光谱反射率校正板数据
           └── 映射 (Mapping)  —— 任务与地块的多对多关联
                └── 地块 (Plot) —— 例如 "小区A3-处理组1"
                     ├── RGB 图像
                     ├── 高光谱数据 (.hdr + .spe)
                     ├── LiDAR 点云 (.ply)
                     ├── 热红外数据 (.ddt + .bmp)
                     └── 处理参数 (Param)  —— 指数均值、模型预测结果等
                          └── 分层数据 (Slice) —— LiDAR 点云切层面积
```

## 支持的数据类型与处理能力

### 1. RGB 可见光图像

- **植被覆盖度 (FVC)**：基于 HSV 色彩空间的植物分割算法，自动提取绿色植被区域，计算覆盖百分比
- 前端通过 Fabric.js 提供图像标注能力

### 2. 高光谱成像 (HSI) —— 核心模块

高光谱是本系统功能最丰富的模块，支持完整的处理流水线：

- **亮度校正**：去除暗电流和高亮过曝像素，使用自适应阈值
- **反射率校正**：基于 3%/30% 标准反射板的两点线性校正，结合光谱插值获取全波段 k、b 系数
- **光谱曲线查看**：点击图像任意像素，实时返回该点的连续光谱反射率曲线（~300 波段）
- **植被指数计算** —— 支持 **30+** 种植被指数，包括：

   | 类别 | 指数 |
   |---|---|
   | 绿度指数 | NDVI, GNDVI, SAVI, OSAVI, EVI, ARVI, MSR, RVI, SLAVI, VARI, DVI, TVI, MTVI2 |
   | 光化学指数 | PRI, PHRI, PSRI, CR1, CRI2 |
   | 色素指数 | ARI1, ARI2, GI, NPCI, MCARI, TCARI, RVSI, YI |
   | 水分指数 | WBI, NRI, SR |
   | 其他 | PSSRa, PSSRb, RVI |

  每种植被指数均支持在线伪彩图渲染和统计均值计算。

- **表型模型预测**：基于 PLS-R（偏最小二乘回归）的模型预测，训练数据为 `TrainData.csv`，可预测 SPAD（叶绿素相对含量）、氮含量（N）、钙含量（Ca）、碳含量（Cb）等生理生化指标，逐像素预测并生成伪彩分布图

- **批量处理**：支持任务级别的批量亮度校正 → 反射率校正 → NDVI 去背景的全自动流水线

### 3. 激光雷达 (LiDAR) 点云

- 原始 `.txt` 数据自动转换为标准 `.ply` 格式
- 基于 **Open3D** 的点云处理：
  - **点云裁剪**：沿 x/y/z 轴方向裁剪
  - **体素下采样**：降低点云密度以提高处理速度
  - **DBSCAN 聚类去噪**：自动过滤离群点
  - **株高计算**：基于百分位数法（如去掉顶部 5% 和底部 5% 后计算高差）
  - **冠幅宽度计算**：基于 y 轴方向百分位数
  - **逐厘米切层**：从地面到冠层顶部按 1cm 间隔切层，计算每层投影面积
- 前端使用 **Three.js** 实现点云的 3D 交互式渲染

### 4. 热红外成像

- 集成 **ThermoGroupSDK**（红外热像仪 SDK），通过 ctypes 调用 C++ 动态库
- 支持 `.ddt` 原始温度数据和 `.bmp` 伪彩图的上传与解析
- **温度提取**：支持单点测温和矩形区域测温，返回区域内最高/最低/平均温度
- 支持跨平台（Windows `.dll` / Linux `.so`）

## 数据采集方式

### Web 端直接上传（Socket.IO 实时通信）

前端通过 Socket.IO 与后端建立长连接，实现大数据量的实时传输。上传管道包含校验环节：

1. 校验用户名是否存在
2. 校验/创建项目文件夹
3. 校验/创建任务文件夹
4. 校验/创建地块文件夹
5. 按数据类型写入文件并入库

### 桌面采集端（WPF 上传器）

针对田间批量采集场景，开发了独立的 Windows 桌面程序：

- 支持读取 CSV/Excel 格式的采集清单
- 自动解析文件路径并按层级关系打包上传
- 通过 SocketIO.Client 与服务器通信
- 可配置服务器地址，支持局域网和远程部署

## 技术亮点

1. **多模态融合**：四种传感器数据在统一的数据模型下管理，通过 Mapping 表实现了任务-地块级别的多模态关联

2. **完整的高光谱处理链**：从原始 DN 值 → 亮度校正 → 反射率校正 → 植被指数 → PLSR 模型预测，全部在线完成，无需借助 ENVI 等专业软件

3. **大文件传输**：HTTP 缓冲区上限 1GB，Socket.IO 支持分块传输，满足高光谱数据（单文件可达数百 MB）的上传需求

4. **生产级部署**：后端通过 Gunicorn + Eventlet 支持 WebSocket 的并发处理，前端独立构建部署

## 总结

mctpweb 是一个面向作物表型研究的综合性 Web 平台，打通了从田间多模态数据采集到云端在线分析的完整链路。系统已在实际的小麦、茶树等作物试验中投入使用，为高通量表型分析提供了高效的工具支撑。未来计划进一步优化异步批处理的响应速度，扩展更多机器学习模型，并增强移动端的适配能力。
